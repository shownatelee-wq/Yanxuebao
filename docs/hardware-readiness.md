# 真机待对接清单

日期：`2026-04-09`

当前软件项目已经完成设备端 H5 / WebView 外壳与正式基础流程，但以下能力仍依赖硬件团队文档或真机接口。

## 登录与设备身份

- 人脸识别登录 SDK
- 设备唯一标识获取方式
- 授权码与设备码的最终映射关系
- 租赁模式 / 销售模式在真机上的行为差异

## 多媒体能力

- 摄像头调用与权限模型
- 录音 / 播放接口
- 文件系统与媒体存储路径
- 本地缓存清理策略

## 交互与系统能力

- NFC 碰一碰
- PTT 物理按键事件
- 通知权限与推送唤醒
- 开机自启、前台驻留、后台保活

## 安全与位置

- GPS / 北斗定位精度与上报周期
- SoS 真实报警链路
- 紧急联系人消息联动
- 远程清理与设备停用

## 软件团队已预留内容

- 设备端授权登录流程
- SoS 页面与交互预留
- 团队、任务、成长、AI、我的等正式流程页
- 演示增强页，用于向硬件团队说明待对接能力边界
- 设备端模式标记：`API 模式 / 演示模式`
- 摄像头 / 录音 / GPS / NFC / PTT / 通知 / 保活 的占位交互入口
- `device bridge` 占位层，已接入人脸登录演示页与 SoS 求助页

## 软件团队下一步接桥清单

1. 基于硬件接口文档补一层 `device bridge`，统一封装人脸、拍照、录音、定位、PTT 和 NFC。
2. 保持 H5 演示页不变，先用桥接层替换当前 mock / 说明逻辑。
3. 先打通 `登录 -> 任务采集 -> 上传 -> SoS` 四条高优链路，再进入社交、会议、网盘等增强能力。

## 下一步建议

- 硬件团队输出《Android 能力接口文档》
- 软件团队基于文档补一层 `device bridge`
- 将当前 H5 演示层逐步替换为真机能力调用层
