# 研学宝 V1 导师端 H5 PRD

## 1. 文档说明

- 文档目标：定义导师端 H5 一期全量原型需求，作为纯前端本地数据原型开发依据。
- 需求来源优先级：用户提供 PDF《研学宝V1开发需求260323》`九、导师端小程序` 为主，仓库内 `研学宝V1_PRD.md` 与 `研学宝V1_WBS.md` 仅作为补充校准。
- 本轮交付边界：
  - 仅开发导师端 H5。
  - 仅使用前端本地数据，不接后端。
  - 页面以微信 H5 风格的手机端工作台实现。
  - 不改动设备端代码。

## 2. 产品定位

导师端 H5 面向学校、旅行社、景区等研学机构导师与助理老师，用于组织研学团队、管理小组、配置与下发任务、查看学员与小组进度、进行评分评价、生成报告、发送广播并处理安全事件。

导师端是研学活动执行期间的移动工作台，强调：

- 手机端快速操作
- 当前团队上下文连续工作
- 任务、作品、评分、报告闭环
- 广播、位置、SoS 的现场管理能力

## 4. 用户与角色

### 4.1 导师

- 可查看系统分配团队与自建团队
- 可切换当前团队
- 可创建与编辑团队、小组、任务
- 可查看作品、评分、评价、报告、位置、SoS
- 可发送团队/小组/学员广播

### 4.2 助理老师

- 只可查看导师授权团队
- 可参与成员配置、任务执行、评分辅助、照片管理
- 不具备创建团队的主权限

本轮原型中默认使用“导师”身份登录，助理权限通过页面内说明与本地数据体现，不额外做权限系统。

## 5. 信息架构与路由

### 5.1 默认流转

1. `/login` 模拟登录
2. 登录成功进入 `/dashboard`
3. 如无当前团队，主面板提示切换或创建团队
4. 导师从主面板或底部导航进入二级模块

### 5.2 页面结构

- `/login` 登录页
- `/dashboard` 导师主面板
- `/teams` 团队管理
- `/tasks` 任务管理
- `/works` 执行工作区（学员 TAB / 小组 TAB / 作品 TAB）
- `/me` 我的
- `/groups` 小组管理详情
- `/ranking` 排行榜详情
- `/reports` 研学报告与成长值详情
- `/broadcasts` 广播与消息详情
- `/photos` 团队照片详情
- `/safety` 位置与 SoS 详情

### 5.3 底部导航

- 工作台：主面板与现场总览
- 团队：团队、小组、助理、资料协同
- 任务：学员任务与小组任务管理
- 执行：学员 TAB / 小组 TAB / 作品 TAB 以及排行、评分、报告联动
- 我的：导师信息、常用工具、重置与退出

### 5.4 全局交互原则

- 始终显示“当前团队”上下文
- 采用单栏卡片式布局，不使用桌面后台侧栏
- 高风险操作采用底部弹层或确认弹窗
- 长列表采用卡片分组、筛选条和折叠展开
- 关键页面提供“一键重置示例数据”

## 6. 全局数据模型与本地数据规则

### 6.1 核心数据对象

- `TutorSession`：登录态、导师信息
- `CurrentTeam`：当前工作团队
- `Team`：团队信息、状态、来源、目的地、资料、助理老师
- `Student`：学员信息、家长信息、设备状态、所在团队/小组
- `Group`：小组、队名、队徽、成员、岗位
- `StudentTask` / `GroupTask`：任务定义、任务说明、附件、作品要求、状态、排序
- `WorkItem`：作品提交、AI 评分、导师评分、评分状态
- `ScoreRecord`：评分记录
- `RewardPenaltyRecord`：奖惩分记录
- `EvaluationItem` / `EvaluationResult`：评价维度、星级结果、综合评级
- `StudyReport`：报告状态、评级、成长值、推送时间
- `BroadcastMessage`：广播范围、内容类型、发送时间
- `PhotoAsset`：团队照片、关联对象、标签
- `LocationSnapshot`：学员位置、地址、距离
- `SosAlert`：SoS 时间、内容、位置、处理状态

### 6.2 本地数据原则

- 所有操作在浏览器本地可即时生效
- 支持本地持久化，刷新后保留
- 支持重置回种子数据
- 新建团队后可继续创建小组、任务、报告
- 新建任务时自动生成对应作品记录
- 评分、奖惩分、评价与报告会联动更新统计结果

### 7.6.2 小组 TAB

#### 列表字段

- 小组序号
- 小组名称
- 小组人数
- 已完成任务数
- 完成进度
- 小组得分

#### 详情功能

- 查看小组任务进度
- 对小组任务评分
- 发放小组奖励分 / 处罚分

### 7.6.3 作品 TAB

#### 列表字段

- 所属对象：学员 / 小组
- 学员或小组名称
- 任务名称
- 提交时间
- 评分状态：未评分 / AI 评分 / 评分确认
- AI 分
- 导师分

#### 操作

- 单个评分
- 批量评分
- 批量确认 AI 分
- 查看作品详情

### 评分规则

- 星级评分 1-5 星
- 0.5 为最小步进
- 分值按任务总分映射计算
- 已有 AI 分时可直接确认 AI 分作为最终分

### 奖惩分规则

- 可对学员或小组单独加分/扣分
- 记录分值、原因、时间、操作人

### 综合评级

- 根据评价项目的导师评分自动计算综合评价分
- 评级规则：
  - `>=95`：A
  - `>=85`：B
  - `>=70`：C
  - `>=50`：D
  - `<50`：E

### 状态与异常态

- 无作品提交
- 无待评分记录
- 当前团队无小组任务
